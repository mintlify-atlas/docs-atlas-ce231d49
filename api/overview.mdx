The Kiosplay API is a JSON REST API that powers the Kiosplay digital signage platform. All endpoints are unauthenticated — no API keys or tokens are required. The API exposes resources for managing kiosks, playlist items, media conversion tools, and a public player data feed.

## Base URL

In local development the server runs on port `3001`:

```
http://localhost:3001
```

All endpoints are prefixed with `/api`. For example, to list all kiosks you call `http://localhost:3001/api/kiosks`.

## Authentication

This is an open API. No authentication is required for any endpoint.

## Request format

Endpoints that accept a request body expect either JSON (`Content-Type: application/json`) or `multipart/form-data` for file uploads. The required content type is noted on each endpoint page.

## Response format

All responses are JSON. Successful responses return the requested resource or a confirmation object. Error responses always follow this shape:

```json
{ "error": "Human-readable error message" }
```

## HTTP status codes

| Code | Meaning |
|------|---------|
| `200` | OK — request succeeded, resource returned |
| `201` | Created — resource created successfully |
| `400` | Bad Request — invalid or missing parameters |
| `404` | Not Found — resource does not exist |
| `500` | Internal Server Error — unexpected server-side failure |
