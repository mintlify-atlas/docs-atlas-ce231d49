This guide takes you from zero to a running kiosk in about five minutes. By the end you will have the admin panel open in your browser, a test kiosk created, and a fullscreen player URL ready to load on any display.

## Before you start

Make sure you have Node.js v18 LTS or later and Git installed. Run `node -v` to check. No other tools are required — Kiosplay uses SQLite, so there is no database server to install.

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone Kiosplay and enter the project directory.

    <CodeGroup>

    ```bash Linux / macOS
    git clone <repo-url> kiosplay
    cd kiosplay
    ```

    ```bash Windows
    git clone <repo-url> kiosplay
    cd kiosplay
    ```

    </CodeGroup>
  </Step>

  <Step title="Install all dependencies">
    A single script installs packages for the root workspace, the server, and the client in one step.

    ```bash
    npm run install:all
    ```

    This is equivalent to running `npm install` in the root, `server/`, and `client/` directories separately. You do not need to `cd` into any subdirectory.
  </Step>

  <Step title="Configure environment variables">
    Copy the example file and place it inside `server/`.

    <CodeGroup>

    ```bash Linux / macOS
    cp .env.example server/.env
    ```

    ```bash Windows
    copy .env.example server\.env
    ```

    </CodeGroup>

    The default values work for local development without any edits:

    ```env
    PORT=3001
    SQLITE_PATH=./data/kiosko.db
    UPLOAD_DIR=./uploads
    PUBLIC_BASE_URL=http://localhost:5173
    MAX_FILE_SIZE_MB=200
    ```

    See [Configure Kiosplay](/configuration) for a full description of each variable.
  </Step>

  <Step title="Start the development servers">
    From the project root, run:

    ```bash
    npm run dev
    ```

    This uses `concurrently` to launch two processes in parallel:

    | Process | URL | Description |
    |---------|-----|-------------|
    | Backend | `http://localhost:3001` | Express API + SQLite, auto-reloads with nodemon |
    | Frontend | `http://localhost:5173` | Vite dev server with hot module replacement |

    The Vite dev server proxies `/api/*` and `/uploads/*` to the backend, so there is no CORS configuration needed in development.

    <Note>
      The terminal prefixes server logs with `[0]` and Vite logs with `[1]`. If only one set of logs appears, check that ports 3001 and 5173 are not in use by another process.
    </Note>
  </Step>

  <Step title="Create your first kiosk">
    Open the admin panel at `http://localhost:5173/admin`.

    1. Click **New kiosk** and give it a name (e.g. "Reception").
    2. Kiosplay automatically generates a URL-safe slug (e.g. `recepcion`).
    3. Open the kiosk editor and upload one or more images or videos.
    4. Set a display duration in seconds for each image.

    Your kiosk player is now live at `http://localhost:5173/recepcion` (replace `recepcion` with your slug). Open that URL in a browser to see the playlist loop in fullscreen.
  </Step>
</Steps>

## Development URLs

| URL | Description |
|-----|-------------|
| `http://localhost:5173/admin` | Admin panel — manage all kiosks |
| `http://localhost:5173/admin/kiosk/:id` | Playlist editor for a specific kiosk |
| `http://localhost:5173/:slug` | Fullscreen player (e.g. `/recepcion`) |
| `http://localhost:3001/api/kiosks` | REST API — direct access, bypasses frontend proxy |

## Available npm scripts

| Script | Description |
|--------|-------------|
| `npm run install:all` | Install dependencies for root, server, and client |
| `npm run dev` | Start server and client in parallel |
| `npm run dev:server` | Start only the Express server with nodemon |
| `npm run dev:client` | Start only the Vite frontend |
| `npm run build` | Build the frontend for production |
| `npm start` | Start the server in production mode |
